# SimpleNodeEditor 設計ドキュメント

## 1. 概要 (Overview)
SimpleNodeEditor は、特定のバックエンド（処理エンジン）に依存しない、**汎用的なAsset駆動型ノードエディタ**です。
本エディタは「処理の実行」を行わず、「データフロー（パイプライン）を視覚的に定義し、JSON（中間表現）として入出力すること」のみを責務とするピュアなオーサリングUIとして機能します。

## 2. コアアーキテクチャ (Core Architecture)

### 2.1 Engine-Agnostic (バックエンド非依存)
エディタ自身は「ノードがどのような処理をするのか」を知りません。
エディタが扱うのは以下の概念のみです。
- **Node (ノード)**: 入出力ポートとパラメータ（プロパティ）を持つ視覚的ブロック。
- **Asset Key (アセットキー)**: ノード間を行き来するデータの識別子（文字列）。

### 2.2 Asset-Driven Connections (アセットキーによる結線)
一般的なノードエディタが `NodeB.input = NodeA.output()` のような直接参照を持つのに対し、本エディタでは「アセットストア」の概念を採用しています。

- **ワイヤーの正体**: UI上でノードAの出力ポートからノードBの入力ポートへワイヤーを繋ぐと、一意な文字列（例: `asset_1a2b3c`）が生成されます。
- **バインディング**: 生成されたキーは、ノードAの `outputs` とノードBの `inputs` の両方に記録されます。
- これにより、エディタはデータの型や中身を解釈することなく、単純な文字列の共有・ライフサイクル管理のみで複雑なデータフローを表現できます。

### 2.3 Schema-Driven UI (スキーマ駆動のUI生成)
エディタのUI（パレット、ノードの見た目、ポートの数、入力フォーム）は、外部ファイル（`schema.json`）によって完全に動的生成されます。

- **schema.json の役割**: 各ノードの `type`、`inputs`（入力の型）、`outputs`（出力の型）、`parameters`（パラメータのUI定義や初期値）を定義します。
- JavaScriptのコアコードを変更することなく、このJSONを書き換えるだけで新しいノードや機能をエディタに追加できます。

### 2.4 Intermediate Representation (中間表現の出力・入力)
エディタは視覚的なグラフ状態を、バックエンドエンジンが実行可能なフラットなJSON形式（IR）へと変換します（`exportIR`）。
また、このJSONを読み込んでUI上のグラフを復元する機能（`importIR`）も備えています。

#### JSONの構造例:
```json
{
  "nodes": [
    {
      "id": "node_1",
      "type": "SpectrumNode",
      "inputs": {
        "in": "asset_x9yz"
      },
      "outputs": {
        "out": "asset_q1w2"
      },
      "parameters": {
        "fft_size": 2048
      },
      "ui": {
        "x": 250,
        "y": 100
      }
    }
  ]
}
```
